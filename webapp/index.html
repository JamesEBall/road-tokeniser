About

This visualisation identifies road segments where the posted speed limit is misaligned with the Safe System recommended speed, given the road's geometry, vulnerable road user (VRU) exposure, and crash history where available.

Each segment is a 5–35 m token derived from OpenStreetMap road geometry, enriched with VRU proximity features and severity-weighted crash counts. The Safe System speed is derived from a rule cascade (motorway pass-through, VRU exposure, junction proximity, curvature, road class). The ML misalignment signal is unsupervised: a Graph Attention encoder pretrained with masked-feature SSL produces an embedding per token, and the score is the distance between the posted limit and the modal posted limit of the 50 geometrically nearest segments in embedding space.

Rendering: vector tiles via MapLibre GL JS + PMTiles. Cambridge ~4 MB, Wellington ~25 MB, Manila ~75 MB — all served as a tile pyramid so the browser only fetches what's visible at the current zoom.

Source: github.com/JamesEBall/road-tokeniser
